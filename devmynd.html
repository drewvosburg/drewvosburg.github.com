Actors Studio

Design Assignment for DevMynd

Actors Studio is a fictional premiere training school for the performing arts looking to supplement their curriculum by incorporating technology into their training methodology.

The feedback process laid out in this assignment is meant to carefully guard against harsh criticism while providing specific, constructive, actionable criticism that can help performers take their craft to the next level.

Giving feedback to performance is a delicate thing. Students are often forming a work in progress, and judgment can do more harm than good if not communicated properly. The feedback process laid out in this assignment is meant to carefully guard against harsh criticism while providing specific, constructive, actionable criticism that can help performers take their craft to the next level.

Assumptions

For the purpose of the assignment, I made a number of significant assumptions. The primary assumption I made was that feedback is for the benefit of the students to practice giving feedback, as opposed to giving instructors insight into performance. It could be that instructors would like an objective rating of each student which is an average of ratings by other performers in the cohort, but I would assume this isn't the case.

A second important assumption is that feedback is meant to benefit the performer receiving the feedback by giving actions that can help them improve, not by judging their current ability or most recent performance. Seeing a numeric rating of your performance could be emotionally damaging and drive down usage of the platform, so instead the focus is providing positive steps toward a better performance.

Research and Sources

I briefly and casually interviewed two friends who have done professional performing arts to get some initial feedback and insights. As a result, I used Liz Lerman's Critical Response Process as a basic framework for my feedback flow, and I used Winthrop University's theater and dance rubric as a reference for my feedback rubric.

Design Considerations

For platform, I assumed a responsive web platform, with a focus on mobile in an effort to get feedback from students as quickly as possible. I also structured the system to use curiousity as a carrot. Performers must input feedback in order to view the feedback that has been given to them. This is meant to incentivize giving feedback without offering money or course credits. It should be noted that getting the process started would be critical for a system like this, so a small prize may be necessary for the first student to give feedback for the first several weeks.

Process Outline

Here is an outline of the process I would use to get from start to finish on a project like this if it were run at scale with a client:

  • Interview instructors and students to find needs and concerns
  • Create detailed personas
  • Map out user flows
  • Create low fidelity clickable prototypes
  • User test prototype with students to get feedback
  • Revise user flows
  • Brand survey
  • Style Guide
  • High fidelity clickable prorotype
  • Full branding (if within the scolpe of the project)
  • Build front end HTML/CSS/JavaScript
  • Build backend database
  • Roll out limited trial to a few instructors/cohorts
  • Revise as needed
  • Deploy at scale
